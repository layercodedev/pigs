## Per-VM Console Output Buffering

Added per-VM output buffering so console output is preserved when switching between VMs. Previously, switching to a different VM cleared the terminal and all output was lost — a critical problem when orchestrating multiple parallel agents. Now each VM maintains its own output buffer (up to 5000 lines), and switching back to a VM restores its full console history. Background VMs continue capturing output even when not displayed.

Changes:
- `src/output-buffer.ts`: New module — per-VM output ring buffer with append, get, clear operations. Splits on newlines, handles \r\n, trims to 5000 lines max per VM.
- `src/tui.ts`: Added `restoreTerminal()` function that sets terminal content from buffered lines and scrolls to bottom. Exported from `createApp()`.
- `src/index.ts`: Integrated output buffer — `connectSessionOutput` now buffers all output per-VM and is idempotent (won't add duplicate listeners). Activate handler restores buffered output instead of clearing. Prompt-submit clears buffer for fresh output. Delete handlers clean up buffers. No longer detaches from other VMs so background output keeps flowing.
- `src/__tests__/output-buffer.test.ts`: 10 tests covering append, trailing newlines, \r\n handling, per-VM isolation, MAX_LINES trimming, unknown VM default, clear single, clear non-existent, and clear all.

All 162 tests pass. Type checks clean.

## Elapsed Time Tracking Feature

Added per-VM elapsed time tracking that shows a live-updating timer in the sidebar when an agent is working on a task. Essential for parallel agent orchestration — see at a glance how long each agent has been running, identify stalled agents, and understand task performance across your fleet. Timer starts when a prompt is sent (single or broadcast), clears automatically when the agent finishes (attention signal fires), and updates every second.

Changes:
- `src/types.ts`: Added `taskStartedAt?: number` field to VM interface
- `src/tui.ts`: Added `formatElapsed()` pure function (compact format: 5s, 1m30s, 1h05m). Updated `renderSidebar()` to display elapsed time in cyan next to VMs with active tasks. Added 1-second interval timer to refresh sidebar when any VM has an active task. Updated help screen with elapsed time info (increased dialog height to 31).
- `src/index.ts`: Set `vm.taskStartedAt = Date.now()` on prompt-submit and broadcast-submit when sending `claude -p` to a VM.
- `src/notification-monitor.ts`: Clear `vm.taskStartedAt` when done-signal is detected (task finished).
- `src/__tests__/format-elapsed.test.ts`: 10 tests covering undefined input, same time (0s), seconds-only, minutes+seconds, hours+minutes, negative elapsed clamping, boundary values, and zero-padding.
- `src/__tests__/types.test.ts`: Added tests for taskStartedAt field (set and undefined).
- `src/__tests__/notification-monitor.test.ts`: Added test verifying taskStartedAt is cleared when signal file is found.

## Stop/Cancel Agent Feature

Added stop/cancel agent (press `x`) to send Ctrl-C to a running agent without attaching to the console. Essential for parallel agent orchestration — when an agent is stuck, going down the wrong path, or you need to interrupt it, press `x` to stop it instantly from the sidebar view instead of having to attach to the console first. Automatically clears the elapsed timer. If no console session exists, one is attached first so the interrupt signal can be delivered.

Changes:
- `src/tui.ts`: Added `x` keybinding in normal mode that triggers `stop-agent` handler. Updated help screen with new keybinding (increased dialog height to 32). Updated status bar to include `x:stop`.
- `src/index.ts`: Added `stop-agent` handler that sends Ctrl-C (`\x03`) to the selected VM's console stdin. Attaches a console session if none exists. Clears `vm.taskStartedAt` to stop the elapsed timer.

All 179 tests pass. Type checks clean.
