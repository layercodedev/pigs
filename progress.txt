## Migrate codebase to Bun

- Updated package.json: scripts use `bun` instead of `node`/`tsc`, entry points point to `src/index.ts` (no build step), removed `typescript` and `vitest` devDeps, added `@types/bun`
- Updated tsconfig.json for Bun: ESNext target/module, bundler moduleResolution, allowImportingTsExtensions, noEmit, bun-types
- Changed shebang from `#!/usr/bin/env node` to `#!/usr/bin/env bun`
- Updated all `.js` import extensions to `.ts` across source and test files (Bun runs TS directly)
- Removed `dist/` directory (no longer needed — no build step)
- Removed `package-lock.json` (replaced by `bun.lock`)
- Removed vitest config files (vitest.config.ts, vitest.integration.config.ts)
- Simplified `.gitignore` (removed dist/ and tsbuildinfo entries)
- Installed Bun 1.3.8 and `@types/bun`
- Verified: typecheck passes, app starts, 259/265 tests pass (6 failures are vitest-specific mocking APIs that need bun:test migration)
