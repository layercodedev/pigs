## 2026-02-06 - Iteration 3: SSH Console Session

Task: Attach to active VM console via sprites exec TTY, pipe stdin/stdout, detach on switch

Completed:
- Created src/console-session.ts module managing TTY sessions via Sprites SDK
  - `attachConsole()` spawns bash with TTY mode on a Sprite VM via `sprite.spawn('bash', [], { tty, cols, rows })`, starts the WebSocket connection, and caches the session
  - `detachConsole()` removes stdout/stderr listeners without killing the remote process (session persists)
  - `destroyConsole()` kills the remote command and removes the session
  - `writeToConsole()` pipes data to the remote stdin
  - `resizeConsole()` sends terminal resize to the remote TTY
  - `detachAll()` gracefully detaches all sessions (used on quit)
  - `getSession()` / `getActiveSessionNames()` for session introspection
- Updated src/types.ts: added 'console' mode to AppState.mode union
- Updated src/tui.ts:
  - Added terminal output box inside main view for displaying console session output
  - Added console mode: raw key input forwarded to VM via `screen.program.on('data')`, Escape detaches
  - Console mode shows yellow border and "(attached)" label, status bar shows "Escape:detach"
  - In console mode, q/Ctrl-C are forwarded to the VM (not intercepted by TUI)
  - Added `getTerminalSize()`, `writeToTerminal()`, `clearTerminal()`, `enterConsoleMode()` methods
  - `resetStatus()` is mode-aware (shows console or normal hints)
  - Screen resize events forwarded to console-resize handler
  - Added 'quit' handler hook for graceful exit
- Updated src/index.ts:
  - Activate handler now creates/reattaches TTY session, connects stdout/stderr to terminal, enters console mode
  - Detaches from previous session's listeners when switching VMs
  - Console-input handler forwards raw keystrokes to VM stdin
  - Console-detach handler (on Escape) detaches session listeners
  - Console-resize handler propagates terminal size changes
  - Delete handler destroys console session before deleting VM
  - Quit handler calls detachAll() for graceful exit
- Created src/__tests__/console-session.test.ts with 16 tests covering:
  - attachConsole: TTY spawn + start, session reuse, multi-VM sessions
  - detachConsole: listener removal, safe no-op for missing sessions
  - destroyConsole: kill + cleanup, graceful error handling
  - getSession: lookup behavior
  - resizeConsole: resize forwarding, safe no-op
  - writeToConsole: stdin write, return value for missing sessions
  - detachAll: multi-session detach
  - getActiveSessionNames: tracking
- All 32 tests pass, typecheck clean, build succeeds

## 2026-02-06 - Iteration 4: VM Provisioning

Task: On create, install Claude Code + SSH via exec, copy CLAUDE.md content from ~/.pigs/settings.json

Completed:
- Created src/provisioner.ts module for VM provisioning via Sprites exec API
  - `loadSettings()` reads ~/.pigs/settings.json, creates default if missing (with default CLAUDE.md content)
  - `provisionVM()` runs provisioning in two steps via `sprite.exec()`:
    1. Installs Claude Code globally (`npm install -g @anthropic-ai/claude-code`) and ensures SSH server is running
    2. Writes CLAUDE.md to /root/ using base64 encoding to avoid shell escaping issues
  - Accepts optional `onLog` callback for progress reporting to TUI status bar
- Updated src/types.ts:
  - Added `provisioningStatus` optional field to VM: 'pending' | 'provisioning' | 'done' | 'failed'
  - Added `PigsSettings` interface with `claudeMd` field
- Updated src/index.ts:
  - Create handler now triggers provisioning after VM creation (runs in background)
  - Shows provisioning progress in status bar via onLog callback
  - Sets provisioningStatus through lifecycle: pending → provisioning → done/failed
- Updated src/tui.ts:
  - Sidebar VM cards now show provisioning status labels: [setup], [wait], [fail]
- Created src/__tests__/provisioner.test.ts with 9 tests covering:
  - loadSettings: reading existing file, creating defaults when missing, valid JSON output
  - provisionVM: exec calls for provisioning script, base64-encoded CLAUDE.md writing, onLog callbacks, error propagation, no-callback usage, default settings creation
- Updated src/__tests__/types.test.ts with 3 new tests for provisioningStatus and PigsSettings
- All 44 tests pass, typecheck clean
