## Bulk VM Deletion Feature

Added bulk delete (press `D` / Shift-D) to delete all VMs at once with confirmation. Essential complement to bulk create — tear down an entire fleet of agents in one action instead of deleting one by one. Deletes happen in parallel with live progress, unmounts filesystems and destroys console sessions before deletion.

- `src/index.ts`: Added `delete-all` handler that unmounts, destroys consoles, and deletes all VMs in parallel via `Promise.all` with progress tracking (`Deleted X/N...`). Tracks success/failure per VM using a Set.
- `src/__tests__/types.test.ts`: Added test for confirm-delete-all mode

All 115 tests pass. Type checks clean.

## Re-provision VM Feature

Added re-provision (press `r`) to update the selected VM's CLAUDE.md and hooks config from disk, and re-provision all (press `R`) to push updated config to every provisioned VM in parallel. Essential for iterating on agent instructions — edit `~/.pigs/settings.json`, then press `r` or `R` to push changes to your fleet without recreating VMs.

Changes:
- `src/types.ts`: Added `'confirm-reprovision-all'` to AppState mode union
- `src/provisioner.ts`: Added `reprovisionVM()` function — reloads settings from disk and updates CLAUDE.md + hooks (skips expensive install step)
- `src/tui.ts`: Added confirm-reprovision-all dialog (yellow border, 50x7), `r` keybinding for single VM, `S-r` keybinding for all VMs with y/n confirmation, q/Ctrl-C to dismiss dialog. Updated help screen with new keybindings (increased dialog height to 25). Updated status bar to include `r:reprov  R:reprov-all`.
- `src/index.ts`: Added `reprovision` handler that reloads settings and calls `reprovisionVM` on selected VM. Added `reprovision-all` handler that reloads settings once, then re-provisions all provisioned VMs in parallel with progress tracking.
- `src/__tests__/provisioner.test.ts`: 6 tests for `reprovisionVM` covering exec count (no install), settings reload from disk, hooks update, onLog callbacks, error propagation, and silent mode.
- `src/__tests__/types.test.ts`: Added test for confirm-reprovision-all mode.

All 140 tests pass. Type checks clean.

## VM Search/Filter Feature

Added search/filter for the VM sidebar (press `/`) to quickly find VMs by name, label, status, or provisioning status. Essential for managing large fleets — type a search term and only matching VMs are shown. Press `Escape` in normal mode to clear the filter. Navigation with `j`/`k` respects the filtered list so you only move between visible VMs.

Changes:
- `src/types.ts`: Added `'search'` to AppState mode union, added `searchFilter` field to AppState
- `src/tui.ts`: Added `filterVMs()` pure function for case-insensitive matching against name, displayLabel, status, and provisioningStatus. Added search dialog (cyan border, anchored to bottom-left above status bar), `/` keybinding, `Escape` to clear filter in normal mode. Updated `renderSidebar()` to filter displayed VMs and show filter indicator in sidebar label. Updated `j`/`k` navigation to only move between filtered VMs. Updated help screen with new keybindings (increased height to 27). Updated status bar to include `/:search`.
- `src/__tests__/filter-vms.test.ts`: 11 tests covering empty filter, empty list, name matching, displayLabel matching, case insensitivity, status matching, provisioningStatus matching, partial strings, no matches, name fallback, and branch matching.
- `src/__tests__/types.test.ts`: Added `searchFilter` to all AppState tests, added test for search mode.

All 152 tests pass. Type checks clean.
