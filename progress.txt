## 2026-02-10: Task 4 - Add 'open app' command

Added a new 'open app' command (keybinding: `w`) that starts the dev server in a
worktree and opens the browser with a subdomain-style localhost URL for easy
identification of which branch is running in which browser tab.

How it works:
- Pressing `w` on a provisioned branch finds an open port starting from 3000
- Starts `npm run dev` with `PORT=<port>` env var in the worktree's right pane
- Opens the browser after a 2-second delay to `http://branchname-reponame.localhost:port`
- The port is tracked per branch via `devServerPort` on the Branch type

Changes:
- types.ts: Added optional `devServerPort` field to Branch interface

- index.ts: Added `open-app` handler that finds a port, starts dev server, opens browser

## 2026-02-10: Task 5 - Add grid feature

Added a grid view feature (keybinding: `G`) that shows all active agent terminals
tiled in a tmux grid simultaneously. This uses tmux's native pane tiling to create
a real multi-terminal view where users can see all running agents at once.

How it works:
- Pressing `G` creates a new tmux window ("pigs-grid") with one pane per active branch
- Each pane uses `watch` to continuously capture the output of the branch's tmux window
- Panes are automatically tiled using tmux's `tiled` layout for even distribution
- Each pane gets a border title showing the branch name for identification
- Pressing `G`, `q`, or `Escape` closes the grid and returns to the control pane
- Only branches with active tasks (taskStartedAt set) are included in the grid

Changes:
- tmux.ts: Added `createGridWindow()`, `killGridWindow()`, `gridWindowExists()` functions
  that manage a tmux window with tiled panes showing live output from branch windows
- types.ts: Added `'grid'` to AppState mode union type
- tui.ts: Added `G` (shift-g) keybinding for grid toggle, grid mode handling in `q`
  and `escape` handlers, updated help dialog and status bar text
- index.ts: Added `grid-open` and `grid-close` handlers, imported new tmux functions,
  added grid cleanup on quit

All 221 unit tests pass. TypeScript type check passes.

## 2026-02-10: Task 6 - Replace touch-file notification with tmux wait-for

Replaced the file-based polling notification system with tmux's native `wait-for`
channel signaling. This gives instant notification when Claude Code finishes a task
(previously had up to 5 seconds of polling latency).

How it works:
- The Claude Code Stop hook now runs `tmux wait-for -S pigs-done-{hash}` instead
  of `touch /tmp/claude-done-signal-{hash}`
- The monitor spawns `tmux wait-for pigs-done-{hash}` listener processes per branch,
  which block until the channel is signaled — giving instant notification
- After each signal, the listener automatically re-registers for the next one
- A slow 10-second poll remains for git label updates only (not for completion)
- New `registerBranch()` function lets index.ts register listeners for newly
  provisioned branches

Changes:
- notification-monitor.ts: Complete rewrite — replaced file polling with tmux
  wait-for channel listeners. `getSignalPath` → `getChannelName`, `checkSignal`
  removed, `pollAll` replaced with `pollLabels` (git labels only), added
  `waitForBranch()` and `registerBranch()`. Stop hook command changed from
  `touch` to `tmux wait-for -S`.
- index.ts: Import `registerBranch`, call it after each successful provisioning
  (4 locations), extracted `onBranchDone` callback for reuse.
- notification-monitor.test.ts: Updated tests for new tmux wait-for API —
  assertions check for `tmux wait-for -S` instead of `touch`, added
  `getChannelName` tests, updated exports.
- integration.test.ts: Updated `makeStopHookCommand` assertion from `touch`
  to `tmux wait-for -S`.

All 224 unit tests pass. TypeScript type check passes.
