## Migrate codebase to Bun

- Updated package.json: scripts use `bun` instead of `node`/`tsc`, entry points point to `src/index.ts` (no build step), removed `typescript` and `vitest` devDeps, added `@types/bun`
- Updated tsconfig.json for Bun: ESNext target/module, bundler moduleResolution, allowImportingTsExtensions, noEmit, bun-types
- Changed shebang from `#!/usr/bin/env node` to `#!/usr/bin/env bun`
- Updated all `.js` import extensions to `.ts` across source and test files (Bun runs TS directly)
- Removed `dist/` directory (no longer needed — no build step)
- Removed `package-lock.json` (replaced by `bun.lock`)
- Removed vitest config files (vitest.config.ts, vitest.integration.config.ts)
- Simplified `.gitignore` (removed dist/ and tsbuildinfo entries)
- Installed Bun 1.3.8 and `@types/bun`
- Verified: typecheck passes, app starts, 259/265 tests pass (6 failures are vitest-specific mocking APIs that need bun:test migration)

## Migrate tests from vitest to bun:test

- Replaced `import { ... } from 'vitest'` with `import { ... } from 'bun:test'` in all 19 test files
- Replaced `vi.fn()`, `vi.clearAllMocks()`, `vi.spyOn()`, `vi.useFakeTimers()`, `vi.useRealTimers()`, `vi.advanceTimersByTime()` → `jest.*` equivalents from bun:test
- Replaced `vi.mock()` → `mock.module()` (bun:test native module mocking API) in 4 files
- Replaced `vi.mocked(fn)` → `fn as Mock<typeof fn>` type casts in provisioner.test.ts and prompt-history.test.ts
- Fixed `expect(...).resolves.not.toThrow()` pattern in mount-session.test.ts (not supported in bun:test — simplified to `await fn()`)
- Fixed strict type issue in integration.test.ts (`process.env.SPRITES_TOKEN` → non-null assertion)
- Verified: typecheck passes, all 270 unit tests pass with 0 failures

## Add multi-platform one-line installation

- Created `install.sh`: cross-platform installer script (macOS, Linux, WSL) that auto-installs Bun if missing, clones the repo to `~/.pigs/bin/pigs`, installs deps, creates a `pigs` launcher script, and adds it to PATH (supports bash, zsh, fish)
- Updated README.md Installation section: replaced old Node.js/npm instructions with one-line `curl | bash` installer and manual Bun-based install instructions
- Updated README.md throughout: replaced all `npm start`/`npm run` references with `bun start`/`bun run` equivalents (Quick Start, Mobile Access/Termius, VibeTunnel, Development sections)
- Removed Node.js 18+ prerequisite from README (Bun is now the only runtime requirement, and the installer handles it)
- Verified: typecheck passes, all 270 unit tests pass with 0 failures

## Add Linear task viewing

- Created `src/linear-client.ts`: Linear GraphQL API client with `LinearIssue` interface, `fetchMyIssues()` (queries `viewer.assignedIssues` for non-completed/cancelled issues), `renderLinearIssues()` (renders issue list with identifier, title, state, priority, labels), and 30s response cache
- Added `'linear'` mode to `AppState` in `src/types.ts`
- Added Linear overlay to `src/tui.ts`: full-screen scrollable overlay (blue border), `Shift-L` keybinding to open/close, `j/k` scrolling, `r` to refresh, `Escape` to close, updated status bar and help dialog
- Added `linear-open` and `linear-refresh` event handlers in `src/index.ts` with error handling for missing `LINEAR_API_KEY`
- Created `src/__tests__/linear-client.test.ts`: 11 unit tests covering empty state, rendering, selection highlighting, priority labels, label display, title truncation, and issue count footer
- Authentication: uses `LINEAR_API_KEY` environment variable with `Authorization` header to `https://api.linear.app/graphql`
- Verified: typecheck passes, 308/309 tests pass (1 pre-existing integration test failure unrelated to this change)
